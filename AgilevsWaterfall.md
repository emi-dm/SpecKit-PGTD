# Metodologías de Desarrollo de Software: Waterfall vs Agile

## 1. Introducción

En el desarrollo de software existen diferentes enfoques metodológicos que determinan cómo se planifica, implementa y entrega un producto. Dos de los modelos más conocidos son **Waterfall (cascada)** y **Agile (ágil)**.

Estas metodologías se diferencian principalmente en:

- Cómo organizan las fases del proyecto
- Cómo gestionan los cambios
- Cómo se entrega valor al usuario
- Cómo se realizan las pruebas y validaciones

Comprender estas diferencias es fundamental para elegir la estrategia adecuada según el tipo de proyecto.

---

# 2. Metodología Waterfall

La metodología **Waterfall** sigue un enfoque **secuencial y lineal**. El proyecto se divide en fases que se ejecutan una después de otra.

## Fases típicas

1. Análisis de requisitos
2. Diseño del sistema
3. Desarrollo
4. Pruebas
5. Despliegue

Cada fase debe completarse antes de comenzar la siguiente.

## Características

- Requisitos definidos completamente al inicio
- Planificación detallada desde el principio
- Cambios difíciles de incorporar
- Entrega del producto al final del proyecto

## Ventajas

- Alta previsibilidad
- Buena documentación
- Adecuado para entornos regulados

## Desventajas

- Baja flexibilidad
- Feedback del usuario muy tardío
- Riesgo elevado si los requisitos cambian

---

# 3. Metodologías Agile

Las metodologías **Agile** utilizan un enfoque **iterativo e incremental**.

El proyecto se divide en ciclos cortos llamados **sprints**, en los que se desarrolla una pequeña parte funcional del sistema.

## Características

- Desarrollo en iteraciones cortas
- Entregas frecuentes
- Feedback continuo del usuario
- Adaptación constante del producto

## Ventajas

- Alta capacidad de adaptación
- Feedback temprano
- Entrega continua de valor

## Desventajas

- Menor previsibilidad
- Requiere participación constante del cliente
- Puede ser difícil en entornos muy regulados

---

# 4. Ejemplo práctico

Supongamos el desarrollo de una **aplicación móvil para gestionar tareas personales**.

## Con Waterfall

1. Se definen todos los requisitos desde el principio:
   - creación de tareas
   - recordatorios
   - organización por listas
   - diseño de todas las pantallas

2. Se diseña toda la arquitectura del sistema.

3. Se implementa toda la aplicación.

4. Se realizan pruebas al final.

5. Se entrega la aplicación completa.

Si el cliente pide una nueva funcionalidad (por ejemplo **listas colaborativas**), el cambio puede ser costoso porque afecta a fases ya cerradas.

## Con Agile

1. Se define una visión general del producto.

Ejemplo de historias de usuario:

- "Como usuario quiero crear tareas"
- "Como usuario quiero recibir notificaciones"
- "Como usuario quiero organizar tareas por listas"

2. Se desarrollan sprints de 2 semanas.

Ejemplo:

Sprint 1:
- crear tareas
- editar tareas
- eliminar tareas

Sprint 2:
- etiquetas
- modo oscuro

3. Al final de cada sprint se presenta una versión funcional.

4. El cliente da feedback y se reordenan prioridades.

Así el producto evoluciona gradualmente según las necesidades reales.

---

# 5. Comparación resumida

| Aspecto | Agile | Waterfall |
|---|---|---|
| Estructura | Iterativa | Secuencial |
| Cambios | Flexibles | Costosos |
| Feedback | Continuo | Tardío |
| Entrega | Incremental | Final |
| Pruebas | Durante todo el proyecto | Principalmente al final |

Las metodologías Agile permiten adaptarse mejor a entornos cambiantes, mientras que Waterfall resulta útil cuando los requisitos son estables y bien definidos.

---

# 6. Conclusión

No existe una metodología universalmente mejor.

La elección depende de:

- estabilidad de los requisitos
- nivel de incertidumbre del proyecto
- grado de regulación del entorno
- necesidad de flexibilidad

En proyectos de software modernos, especialmente productos digitales, los enfoques **Agile** suelen ser preferidos debido a su capacidad de adaptación y aprendizaje continuo.
